## Who This Guide Is For

This guide is designed for:

- **Site Developers** who configure and customize Quiqr sites
- **Theme Developers** who create Hugo themes for use with Quiqr
- **Content Architects** who design content structures
- **Advanced Users** who want to understand Quiqr's capabilities

## Development Workflow

A typical Quiqr site development workflow:

1. **Plan** - Define content types and structure
2. **Configure** - Create content model with field definitions
3. **Test** - Preview and iterate on the model
4. **Deploy** - Share site configuration with content editors
5. **Maintain** - Update model as needs evolve

## Key Concepts

### Content Model

The content model defines:
- What content types exist (singles, collections)
- What fields each type has
- How content is organized
- UI behavior and validation

### User Roles

Quiqr has two roles:
- **Content Editor** - Simplified interface for content work
- **Site Developer** - Full access to configuration and tools

### Hugo Integration

Quiqr integrates deeply with Hugo:
- Integrated preview server
- Automatic content detection
- Frontmatter management
- Asset handling
